## 🛠️ Key Technical Implementations (Manual Strategy)

### 1. Pure Semantic HTML5 Architecture
The document architecture avoids generic code structures and is strictly categorized under explicit semantic landmark boundaries:
* `<header>` $\rightarrow$ Tracks global structural navbar positioning and fixed top navigation anchoring constraints.
* `<main>` $\rightarrow$ Binds the primary target flow elements safely.
* `<section>` $\rightarrow$ Explicitly groups structural components like `#hero`, `#about`, `#store-features`, and `#products`.
* `<footer>` $\rightarrow$ Manages bottom corporate information layers, legal lists, and navigation blocks.

### 2. Manual Flexbox Layout Framework
Every horizontal grid array on the page was engineered by establishing unique flex formatting contexts:
* **The Navigation Bar:** Handled through `display: flex` combined with `justify-content: space-between` and `align-items: center` to lock elements into fixed edge bounds.
* **The Split Hero Section:** Constructed using responsive flex items that separate text assets cleanly from artwork column panels.
* **Responsive Fluid Wrapping:** Implemented explicit `flex-wrap: wrap` parameters alongside flexible baseline settings (like `flex: 1 1 300px`) across the product display blocks and features lists. This forces child nodes to automatically drop down cleanly on small smartphone displays without needing rigid break lines.

### 3. Absolute Layout Constraints
* Set responsive vertical footprints utilizing standard proportional scaling boundaries (`min-height: 90vh`).
* Utilized native CSS margins and block alignments (`margin: 0 auto`) to keep structural containers centered across large displays.
